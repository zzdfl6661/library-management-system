
# 图书馆管理系统修复建议文档

## 一、问题概述

### 问题一：当月借书学生排行榜和当月热门图书排行榜不能正常显示
- **现象**：首页的两个排行榜区域显示空白，但关闭运行后反而能显示（实际显示的是前端mock数据）
- **影响范围**：首页统计模块

### 问题二：图书检索搜索作者或ISBN无结果
- **现象**：图书列表可正常显示，但搜索作者或ISBN时返回"No Data"
- **影响范围**：图书检索模块

### 问题三：借书证列表中部分字段不显示
- **现象**：借书证号和学生ID列显示为空，只有状态列有值
- **影响范围**：借书证管理模块

---

## 二、问题根因分析

### 问题一：排行榜不显示

**根本原因：后端API路径不匹配 + 数据查询逻辑问题**

1. **API路径不一致**：
   - 前端 `Home.vue` 调用：`/statistics/dashboard`
   - 后端 `StatisticsController` 映射：`/api/statistics/dashboard`
   - 前端 `request.js` 配置了 `baseURL: '/api'`，理论上应该正确拼接为 `/api/statistics/dashboard`

2. **数据查询逻辑缺陷**：
   - 在 `StatisticsServiceImpl.java` 的 `getBookBorrowRanking()` 方法中：
     - 书名显示为 `"Book-" + entry.getKey()` 占位符，未查询实际书名
     - 缺少对当月数据的筛选，查询了所有历史数据

3. **前端fallback机制**：
   - 当API请求失败时，前端会使用mock数据填充，这就是"关闭运行后反而显示"的原因

### 问题二：搜索无结果

**根本原因：多条件搜索逻辑错误**

在 `BookSearch.vue` 的 `handleSearch()` 方法中：

```javascript
const response = await request.get('/books/search', {
  params: {
    title: searchForm.keyword,
    author: searchForm.keyword,
    isbn: searchForm.keyword,
    publisher: searchForm.keyword
  }
})
```

将同一个关键字同时传给了四个参数，而后端 `BookMapper.xml` 的SQL使用 **AND** 连接所有条件：

```xml
SELECT * FROM book WHERE 1=1
<if test="title != null and title != ''">AND title LIKE CONCAT('%', #{title}, '%')</if>
<if test="author != null and author != ''">AND author LIKE CONCAT('%', #{author}, '%')</if>
<if test="isbn != null and isbn != ''">AND isbn LIKE CONCAT('%', #{isbn}, '%')</if>
<if test="publisher != null and publisher != ''">AND publisher LIKE CONCAT('%', #{publisher}, '%')</if>
```

**结果**：只有当书名、作者、ISBN、出版社**同时包含**搜索关键字时才会返回结果，这几乎不可能匹配。

### 问题三：借书证列表字段不显示

**根本原因：数据库字段与实体类属性映射不一致**

查看 `LibraryCardMapper.xml` 的查询：
```xml
SELECT * FROM library_card
```

实体类 `LibraryCard` 的属性：
- `cardNo` - 借书证号
- `studentId` - 学生ID

数据库表字段（根据insert语句推断）：
- `card_no` - 借书证号
- `student_id` - 学生ID

MyBatis默认使用驼峰命名转换（`card_no` → `cardNo`），但需要确认配置是否正确。

## 三、修复方案

### 问题一修复：排行榜不显示

**修改文件**：

1. **后端：`StatisticsServiceImpl.java`** (`/workspace/library-management-system/src/main/java/com/library/service/impl/StatisticsServiceImpl.java`)

   **问题**：`getBookBorrowRanking()` 方法缺少：
   - 当月数据筛选
   - 实际书名查询

   **修复代码**：
   ```java
   @Override
   public List<RankingResponse> getBookBorrowRanking() {
       Map<Long, Integer> bookCountMap = new HashMap<>();
       LocalDate now = LocalDate.now();
       LocalDate startOfMonth = now.withDayOfMonth(1);
       LocalDate endOfMonth = now.withDayOfMonth(now.lengthOfMonth());
       
       List<BorrowRecord> records = new ArrayList<>();
       List<LibraryCard> cards = libraryCardMapper.selectAll();
       for (LibraryCard card : cards) {
           // 添加当月筛选条件
           records.addAll(borrowRecordMapper.selectByCardIdAndMonth(card.getId(), startOfMonth, endOfMonth));
       }
       for (BorrowRecord record : records) {
           BookCopy copy = bookCopyMapper.selectById(record.getCopyId());
           if (copy != null) {
               bookCountMap.merge(copy.getBookId(), 1, Integer::sum);
           }
       }
       List<RankingResponse> ranking = new ArrayList<>();
       bookCountMap.entrySet().stream()
               .sorted((a, b) -> b.getValue().compareTo(a.getValue()))
               .limit(10)
               .forEach(entry -> {
                   RankingResponse r = new RankingResponse();
                   // 查询实际书名
                   Book book = bookMapper.selectById(entry.getKey());
                   r.setName(book != null ? book.getTitle() : "Book-" + entry.getKey());
                   r.setCount(entry.getValue());
                   ranking.add(r);
               });
       return ranking;
   }
   ```

2. **添加BookMapper依赖**：
   在 `StatisticsServiceImpl` 中注入 `BookMapper`：
   ```java
   @Autowired
   private BookMapper bookMapper;
   ```

3. **添加Mapper方法**：
   在 `BorrowRecordMapper.java` 中添加：
   ```java
   List<BorrowRecord> selectByCardIdAndMonth(@Param("cardId") Long cardId, 
                                              @Param("startDate") LocalDate startDate, 
                                              @Param("endDate") LocalDate endDate);
   ```

   在 `BorrowRecordMapper.xml` 中添加：
   ```xml
   <select id="selectByCardIdAndMonth" resultType="com.library.entity.BorrowRecord">
       SELECT * FROM borrow_record 
       WHERE card_id = #{cardId} 
         AND borrow_date BETWEEN #{startDate} AND #{endDate}
   </select>
   ```

### 问题二修复：搜索无结果

**修改文件**：

1. **前端：`BookSearch.vue`** (`/workspace/library-management-system/frontend/src/views/BookSearch.vue`)

   **方案A**：只传一个参数，后端处理多字段搜索（推荐）
   ```javascript
   const handleSearch = async () => {
     try {
       const response = await request.get('/books/search', {
         params: {
           keyword: searchForm.keyword  // 只传一个关键字
         }
       })
       if (response.code === 200) {
         books.value = response.data
       }
     } catch (error) {
       console.error('搜索失败:', error)
     }
   }
   ```

2. **后端：`BookController.java`** (`/workspace/library-management-system/src/main/java/com/library/controller/BookController.java`)
   ```java
   @GetMapping("/search")
   public ApiResponse<List<Book>> searchBooks(
           @RequestParam(required = false) String keyword) {
       List<Book> books = bookService.searchBooks(keyword);
       return ApiResponse.success(books);
   }
   ```

3. **后端：`BookService.java`** 和 `BookServiceImpl.java`
   ```java
   // BookService.java
   List<Book> searchBooks(String keyword);
   
   // BookServiceImpl.java
   @Override
   public List<Book> searchBooks(String keyword) {
       return bookMapper.searchByKeyword(keyword);
   }
   ```

4. **后端：`BookMapper.java`**
   ```java
   List<Book> searchByKeyword(@Param("keyword") String keyword);
   ```

5. **后端：`BookMapper.xml`**
   ```xml
   <select id="searchByKeyword" resultType="com.library.entity.Book">
       SELECT * FROM book WHERE 1=1
       <if test="keyword != null and keyword != ''">
           AND (title LIKE CONCAT('%', #{keyword}, '%')
                OR author LIKE CONCAT('%', #{keyword}, '%')
                OR isbn LIKE CONCAT('%', #{keyword}, '%')
                OR publisher LIKE CONCAT('%', #{keyword}, '%'))
       </if>
   </select>
   ```

### 问题三修复：借书证列表字段不显示

**修改文件**：

1. **检查MyBatis配置**：
   在 `MyBatisConfig.java` 中确认是否配置了驼峰命名转换：
   ```java
   @Bean
   public ConfigurationCustomizer configurationCustomizer() {
       return configuration -> configuration.setMapUnderscoreToCamelCase(true);
   }
   ```

2. **检查实体类字段**：
   确认 `LibraryCard.java` 的属性名是否正确：
   ```java
   private String cardNo;    // 对应 card_no
   private Long studentId;   // 对应 student_id
   ```

3. **确认数据库表结构**：
   检查 `schema.sql` 中的表定义是否包含这些字段。

## 六、代码优化建议

1. **统一API响应格式**：确保所有接口返回统一的 `ApiResponse` 格式
2. **添加日志记录**：在关键业务逻辑处添加日志，便于排查问题
3. **异常处理增强**：统一异常处理，返回更详细的错误信息
4. **参数校验**：在Controller层添加参数校验注解
5. **分页支持**：对于列表查询接口添加分页功能

## 七、总结

| 问题 | 根因 | 修复位置 |
| :--- | :--- | :--- |
| 排行榜不显示 | 数据查询缺少当月筛选，书名使用占位符 | `StatisticsServiceImpl.java` |
